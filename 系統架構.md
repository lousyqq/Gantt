# 系統架構.md — MSD 專案追蹤總表

## 技術棧

| 層 | 技術 |
|----|------|
| 前端 | React 18（UMD 本地化於 `wwwroot/lib/`，無 CDN）＋ Tailwind CSS；原始碼 `ClientApp/app.jsx` 經 Babel CLI 預編譯為 `wwwroot/app.js` |
| 後端 | ASP.NET Core 9 Minimal API（根目錄 `Gantt.csproj`／`Program.cs`），ADO.NET（Microsoft.Data.SqlClient）直呼預存程序 |
| 資料庫 | SQL Server 2019（開發 `Sariel\Gantt`；遠端正式主機另置）；結構詳見 `DB_table.md` |
| 驗證 | Negotiate（Windows 驗證）僅用於 `/api/whoami` 取工號；其餘端點匿名 |
| 匯出 | ClosedXML 0.105.0（週報／成果清單 .xlsx） |
| 部署 | IIS 子應用程式（如 `/Gantt`）；需啟用 Windows 驗證＋匿名驗證並存 |

## 模組圖與資料流

```
瀏覽器 SPA (app.js, 單一 App 元件)
  │  載入時: GET /api/whoami (Negotiate→工號) → GET /api/access-check (瀏覽權限, fail-closed)
  │          → GET /api/bootstrap?year= (一次載入全部資料) → 60 秒靜默輪詢 refreshData
  │  寫入:   POST /api/*  (apiPost 統一自動附 actorEmpId)
  ▼
Minimal API (Program.cs)
  │  讀取: bootstrap / audit-log(白話翻譯在此層) / login-stats / access-check(查 notes_person)
  │  寫入: 一律 CommandType.StoredProcedure；錯誤統一 Fail(ex)——RAISERROR(50000)照原文回400、其餘記log回一般化500
  │  連線字串 ConnStr() 每次即時讀 appsettings (reloadOnChange，改檔即生效)
  ▼
SQL Server（24 個 SP，全部寫 AuditLog；權限檢查在 SP 內做：主管限定/負責人限定）
  └─ 跨 server VIEW [WEB].[dbo].[notes_person]（部門名冊，瀏覽權限比對用；本機以 sim 腳本模擬）
```

## 前端模組（app.jsx 內單檔元件）

- **三種檢視**（viewMode）：週檢視（打卡操作）／年度總覽（整年一頁唯讀）／成果清單 ResultsView（高階主管唯讀＋★重點關注＋排序篩選＋Excel 匯出）。
- **成員回報動線**：header「📋 本週回報中心」PendingPanel（進度條＋任務清單＋下週預計必填＋非專案選填＋Completion Loop）；
  補登開啟時另有「🕘 修改 W{..} 回報」retro 面板。
- **主管功能**：TaskModal 排程編輯／評分、ProjectEditModal（含改負責人）、IntervalModal、DeliverableModal（產出＋MP）、
  CommentModal（週報回覆）、ManagerWeekPanel（代成員修任一週）、補登總開關；
  「⚙️ 管理 ▾」下拉（header 右側帳號區）收納低頻入口：MemberPanel（成員管理）／AccessPanel（瀏覽權限）／
  UsageStatsPanel（使用統計）／AuditPanel（異動紀錄）。
- **共用機制**：ConfirmModal、Toast（❌開頭=錯誤、action 鈕如「↩ 復原」）、MetaLine（最後編輯時間/人）、
  ESC 關窗鏈 closeTopModal、鍵盤導航（H 回本週/←→ 平移）、登入持久化 `gantt_login`、檢視偏好 `gantt_prefs`、
  WeeklyReportDashboard（團隊總結：角色感知＋複製週報＋Excel 匯出）、DeadlinePanel（到期清單）。

## 建置與部署流程

1. 前端：改 `ClientApp/app.jsx` → `npm run build`（產生 app.js/app.css＋`scripts/stamp-assets.js` 自動蓋 `?v=時間戳` 快取破壞）。
2. 後端：`dotnet build Gantt.csproj`；本地執行 `dotnet run --project Gantt.csproj --urls http://localhost:5099`。
3. 發布：`dotnet publish` → IIS 子應用程式（App Pool＝No Managed Code；Windows 驗證＋匿名驗證皆啟用）。
   前端 `API_BASE` 執行期自動偵測子目錄，本地測好可直接發布。
4. DB 變更：新增編號遷移檔（`DB_table.md` 規則），遠端依編號順序執行。
